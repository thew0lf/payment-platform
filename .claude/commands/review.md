# Senior Developer Review

Perform a comprehensive senior developer review of: $ARGUMENTS

## Review Checklist

1. **Code Analysis** - Correctness, edge cases, potential bugs, error handling
2. **Architecture Assessment** - Right approach? Better patterns? Scalability concerns?
3. **Opinion/Critique** - Call out what doesn't look right or feels off
4. **Suggestions** - Improvements, additional work, better UX
5. **Trade-offs** - Pros/cons of current approach vs alternatives
6. **Security Review** - Vulnerabilities, input validation, auth/authz gaps
7. **Performance** - N+1 queries, unnecessary re-renders, caching opportunities
8. **Maintainability** - Code clarity, naming, future developer experience
9. **Testing Gaps** - What should be tested? Edge cases to cover?
10. **Technical Debt** - Shortcuts taken, things to revisit later
11. **Dependencies** - Impact on other systems, breaking changes
12. **Standards Compliance** - Does it follow project conventions?

## Output Format

Provide actionable feedback with:
- Issues found (with severity: critical/major/minor)
- Specific recommendations
- Final verdict: **Approve**, **Approve with suggestions**, or **Request changes**
